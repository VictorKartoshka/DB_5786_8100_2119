## Introduction

This relational database architecture is designed to centralize and manage the core operational data of a modern restaurant. The system records immutable data regarding customer identities, chronological dining reservations, dynamic walk-in waitlists, qualitative dining feedback, and a tiered loyalty rewards ledger. 

The primary functionality ensures that restaurant administration can seamlessly track a patron's lifecycle—from booking a table to accumulating transaction-based loyalty points—while strictly enforcing chronological integrity and mathematical constraints to prevent data anomalies.
